# 分身宠物 · 情侣版

> 你养的宠物,是 AI 扮演的「另一半」。

一个给情侣两个人玩的手机网页小游戏:你们各自捏一只**代表对方**的分身宠物,用它来隔空互动——骂两句、喂狗粮、抱一下、说想你、闹别扭又和好。分身由 AI 驱动,始终在人设里、无厘头、绝不承认自己是 AI;真人也能随时"附身"顶替它说话。核心是把两个人日常那点闹腾、想念、撒糖,变成一只会回话的宠物。

---

## 这是什么

- **一对一配对**:一方创建情侣、拿到邀请码,另一方输码加入,绑成一对。
- **两只镜像分身**:配对后自动生成两只分身——你养的那只代表 TA,TA 养的那只代表你。你在你的分身身上做的动作,会出现在 TA 的时间线里。
- **共享的关系数值**:委屈、狗粮、想你、亲密四个值两人共用,随互动和时间变化,是这段关系此刻的"温度计"。
- **AI 分身 + 真人附身**:平时分身用 AI 按人设回你;真人也能亲自下场,用"本尊回应"顶替 AI 说这句话。

---

## 核心功能

### 🎭 捏分身
给你眼里的 TA 设定一只宠物:**基调**(沙雕 / 傲娇 / 温柔…)、**名字**、**emoji 形象**、一段**种子人设**(TA 是个什么样的人),以及**性别**。这些会揉进 AI 的提示词,决定它怎么跟你说话。

### 💢 七种互动动作
每个动作既改数值、也会引出一句回应:

| 动作 | 数值变化 | 回应来源 |
|---|---|---|
| 骂一顿 `scold` | 委屈 +15 | 🤖 AI 回怼 |
| 戳一下 `poke` | 委屈 +5 | 本地沙雕文案 |
| 喂狗粮 `feed_dogfood` | 狗粮 +20,委屈 −10 | 本地沙雕文案 |
| 抱抱 `hug` | 想你 −30,亲密 +10 | 本地沙雕文案 |
| 说想你 `miss_you` | 想你 −20,亲密 +6 | 本地沙雕文案 |
| 道歉 `apologize` | 委屈 −25,亲密 +8 | 本地沙雕文案 |
| 唠两句 `chat` | — | 🤖 AI 按人设回 |

> 便宜动作走**本地文案**(每个动作 50+ 条,尽量不重样,不烧 API);**骂 / 唠嗑**才叫 AI,省钱又不掉品质。

### 📊 四个关系数值(会自己变)
- **委屈**、**狗粮**、**想你**、**亲密**,范围 0–100。
- **随时间漂移**:想你每小时 +4(越久越想),委屈每小时 −2、狗粮每小时 −1(慢慢平复/消化);**亲密不随时间变**,只靠互动积累。
- **委屈 ≥ 80** 触发系统旁白「该哄了」,提醒你去哄。

### 🤖 AI 回应(DeepSeek 驱动)
分身回话时,提示词会结合:**人设 + 基调 + 性别 + 此刻心情**(某个值 ≥60 就带上对应情绪,比如委屈高就"带刺想被哄")**+ 最近若干条对话上下文**。风格固定为无厘头、1–3 句、口语、绝不承认自己是 AI。
- **API Key 只在服务端**,前端永远看不到。
- **没配 Key 也能玩**:自动走本地沙雕兜底,骂/聊天也有话说(适合离线 / 自测)。
- 每人每天有 AI 次数上限(默认 50 次),防跑量。

### 😽 分身主动撩你
页面开着没人理时,分身会**自己冒泡**来撩你/找话题——结合当前的委屈、狗粮、想你、亲密和 TA 的性别人设生成。空闲一小会儿(默认 55 秒)才触发,不刷屏。有 Key 走 AI,没有就用本地"撩人"文案兜底。

### 💬 统一聊天时间线
事件和聊天合成**一条时间线**:
- 普通动作显示成一行 tip(如「TA 戳了你」),聊天/AI 回应显示成气泡。
- **AI 气泡**和**真人(本尊)气泡**样式区分;说话人按性别上色(男蓝 / 女粉),左右对齐。
- **向上滑动懒加载**:首屏只拉最新一小页,往上滑再补更早的,不会一次性全量加载;新消息实时在底部追加。

### 🎨 移动端 & 性别主题
为手机而做的扁平 UI:`100dvh` 满屏、底部安全区适配、固定底栏和输入框、宠物"呼吸光环"随心情变色。整体主色跟随登录者性别——**男生蓝、女生淡粉**。

### 🛟 稳的那些细节
- **幂等**:连点或断网重试,同一动作对方只会收到一条,数值不会重复涨。
- **断网卖萌**:请求失败时是一句卖萌台词 + 数字不动,网络恢复自动补拉,不会白屏。
- **记住我 / 自动登录**:勾一下下次自动进,刷新不掉登录。

### 🔥 情侣火苗 + 每日一问
- **火苗**:两人**每天都露面**(任一互动或答题)才续上,连续天数越攒越高;一方完成、另一方没动时进入"就差你了"状态(将来接推送召回)。漏一天暂时没得救——花点亲密"续火"的入口还在做,先靠两人自觉。
- **每日一问**:每天一道 AI 混味题(暧昧 / 深度 / 沙雕轮换),**双方都答完才解锁对方答案**——答完在等 TA,就是每天回来的理由。解锁后落进聊天时间线可回看。

## 界面结构(三个页签)

| 页签 | 内容 |
|---|---|
| 🏠 **TA** | 你养的分身:形象 + 光环、四个数值、互动动作按钮。对方还没捏分身时显示「孵化中 + 催 TA」。 |
| 💬 **聊天** | 上面那条统一时间线:动作 tip、AI/真人气泡、主动撩你,底部输入框直接唠。 |
| ⚙️ **我** | 我自己的这只分身与相关设置。 |

## 技术栈

- **前端**:Vite + React 18 + TypeScript,TanStack Query(轮询/缓存),React Router,framer-motion(动效);Vitest + MSW 测试。
- **后端**:FastAPI + SQLAlchemy 2.0,MySQL 8(`utf8mb4`),JWT 鉴权;AI 走 DeepSeek(httpx)。
- **部署**:Docker Compose 三件套 `db` + `backend` + `web`(nginx 托管前端 + 同源反代 `/api`)。

---

## 快速开始

```bash
cp .env.example .env      # 可选:改端口 / 密码 / 填 DeepSeek Key
./start.sh -d             # 打包前端 + 后台起三个容器
```

浏览器打开 **http://localhost**(默认 80 端口)。

- 手机 / 局域网访问、绑定 `.local` 本地域名、当服务器长期运行等,见 **[`deploy/README.md`](deploy/README.md)**。
- 想点亮真 AI:在 `.env` 填 `DEEPSEEK_API_KEY=sk-...` 再 `docker compose up -d backend`;留空则全程走离线沙雕兜底。

> 两个人才玩得出隔空互动的爽点——用两台设备(或两个隐身窗口)各注册一个号走一遍,对照 [`docs/couple-playtest-checklist.md`](docs/couple-playtest-checklist.md)。
