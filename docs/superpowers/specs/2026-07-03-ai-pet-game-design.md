# AI 沙雕宠物养成游戏 · 设计文档

- **日期**：2026-07-03
- **状态**：设计已确认，待进入实现计划
- **一句话**：一款主打休闲、创意、无厘头的手机网页宠物游戏——你养一只被 DeepSeek 驱动、有独特沙雕灵魂的宠物，它会随你的养育方式慢慢变异进化；玩腻了孵新蛋换代，老宠带着 AI 自动写的生平传记进图鉴。

---

## 1. 愿景与核心体验

- **主打**：休闲、创意、无厘头搞笑。不追求精致画面，把资源全砸在"AI 生成的内容够不够骚"上。
- **核心情感钩子**：
  - **短期**——"这货聊天/反应太沙雕了"，被逗笑。
  - **中期**——"它在按我的养法悄悄变异，我好奇它会长成啥怪物"。
  - **长期**——"图鉴越攒越厚 + 想开出更离谱的新宠"。
- **最大风险**：不是技术，而是"AI 生成的内容到底好不好笑、循环黏不黏人"。因此第一版的首要目标是**尽快做出可玩成品去验证乐趣**，而非堆功能。

---

## 2. 核心设计决策（骨架）

| 维度 | 决策 |
|---|---|
| 关系结构 | **一只主宠 + 图鉴收集** |
| 互动方式 | **混合**：照顾按钮养数值 + 聊天养灵魂/搞笑 |
| 性格来源 | **先天 AI 人设 + 后天变异进化**（可黑化/觉醒/叛逆期） |
| 收集机制 | **孵蛋换代**；老宠"毕业"带 AI 生成的沙雕传记进图鉴 |
| 经济系统 | **轻经济**：coins（软通货）+ quests（离谱任务）+ eggs |
| 规模 | 可分享的 Web 小成品（轻后端藏 key + 云存档 + 简单账号） |
| 账号 | 简单账号（昵称 + 密码），可跨设备 |
| 技术栈 | **React（Vite）+ FastAPI + MySQL + DeepSeek** |
| AI 成本 | 每用户**每日 AI 聊天软上限**（~50 条）护钱包 |
| 图鉴老宠 | **只读**：看传记 + 名场面，不可再聊天（升级位保留） |

## 3. 系统架构

四个部分：

- **React 前端（游戏客户端）**：Vite 打包的移动端竖屏单页应用。纯展示层——画面、交互、把操作发给后端、展示状态与聊天。**不碰 DeepSeek key，不写游戏规则。**
- **FastAPI 后端（大脑）**，负责四件事：
  1. **账号**：昵称+密码注册登录，发 JWT。
  2. **游戏状态读写**：宠物数值、性格、图鉴、蛋/道具的增删改查。
  3. **规则引擎**：数值随时间衰减、互动加减、变异/进化触发条件、孵蛋、经济结算——**逻辑全在后端**，前端只显示结果。
  4. **DeepSeek 代理 + Prompt 编排**：生成人设、聊天回复、毕业传记、离谱事件/任务文案。key 藏在服务器环境变量。
- **MySQL**：存用户、宠物、图鉴、聊天记录、蛋/道具。
- **DeepSeek API**：只被 FastAPI 调用，前端永不可见 key。

**关键决定**：
- **规则逻辑全放后端、前端纯展示**（防作弊、单一真相源）。
- **随时间衰减的数值用时间戳懒计算**——每次读取时用"距上次更新过了多久"现算，不跑后台定时任务（省钱省事）。

**一次聊天的数据流**：前端发消息 → `POST /pets/{id}/chat` → 后端读出性格+近期上下文+当前数值，拼 prompt 调 DeepSeek（流式返回做打字机）→ 规则引擎结算效果写回 MySQL → 前端显示回复并刷新数值。

**部署形态**：前端静态托管，后端小容器/服务器，MySQL 托管实例。留好后续加账号体系、加缓存的升级位。

## 4. 数据模型（MySQL）

原则：① AI 驱动、会变形的数据用 **JSON 字段**存（灵活）；② 每只宠物存一份**滚动记忆摘要**控制 token。

### `users`（账号 + 钱包）
`id` · `nickname`(唯一) · `password_hash` · `coins`(INT，软通货) · `created_at` · `last_login_at`

### `pets`（核心表：主宠与图鉴老宠同表，用 `status` 区分）
- `id` · `user_id`
- `name`（孵化时 AI 起的沙雕名，玩家可改）
- `appearance`（JSON：emoji/符号 + 颜色 + AI 生成的外形描述）
- `persona`（JSON：先天人设——性格特质、说话风格、口头禅、身世、怪癖）
- `stats`（JSON：饱食度/心情/清洁/精力/亲密度，各 0–100）
- `stats_updated_at`（时间戳，配合懒计算衰减）
- `evolution`（JSON：**变异影响向量**——记录养育风格，如"辣条 37 次、哲学 12 次、冷落 5 次"，规则引擎据此判断何时/如何进化）
- `memory_summary`（TEXT：AI 定期把久远聊天+大事件压缩成"它记得的事"）
- `status`（`active` 主宠 / `graduated` 图鉴）
- `biography`（TEXT：毕业时 AI 写的沙雕生平，未毕业为空）
- `created_at`(孵化时间) · `graduated_at`

### `chat_messages`（聊天记录）
`id` · `pet_id` · `role`(玩家/宠物/系统事件) · `content` · `created_at`
（只有最近 N 条进 prompt，更久的靠 `memory_summary` 兜底）

### `eggs`（收集入口）
`id` · `user_id` · `rarity`(稀有度，影响生成人设的离谱程度) · `source`(任务奖励/购买) · `hatched`(bool) · `obtained_at`

### `quests`（宠物的离谱要求，驱动日常循环）
`id` · `pet_id` · `description`(AI 现编) · `reward`(coins/蛋) · `status` · `created_at`

## 5. 核心玩法循环

三层：底层"养成数值"给手感目标，中层"聊天灵魂"给惊喜搞笑，顶层"变异进化"给养成回报与收集动力。

### 数值（0–100，规则引擎管）
- **饱食度、清洁度、精力**：随时间衰减（懒计算）。
- **心情**：受其它数值和互动影响。
- **亲密度**：只涨不太掉，是"感情总账"，也是进化/事件门槛之一。

### 中层 · 聊天（灵魂 + 沙雕）
自由打字，宠物用它的性格回你。无厘头笑点主战场，顺带影响进化（聊天话题）与亲密度。

### 底层 · 照顾按钮（手感 + 目标）
点按钮 → 规则引擎即时改数值 + 宠物用符合性格的方式回应：
- **喂食**（+饱食度）：不同食物触发不同反应与**变异倾向**（辣条→暴躁杠精；健康餐→养生怪）。
- **玩耍**（+心情 −精力）、**洗澡**（+清洁，有的宠物死活不肯洗会闹）、**休息**（+精力）。
- 大部分按钮反应走**本地模板+随机**（便宜即时），只有特殊食物/关键时刻才叫 DeepSeek。

### 顶层 · 变异进化（杀手锏）
`evolution` 影响向量默默记录养育风格。规则引擎满足条件时触发**进化事件**，让 DeepSeek 改写人设 + 写夸张沙雕过场。例：
- 长期冷落 → 离家出走/抑郁事件
- 辣条成瘾 → 暴躁杠精觉醒，开始怼你
- 常聊哲学 → 觉醒成话痨哲学家
- 宠爱过度 → 粘人到离谱，夺命连环 call

### 收集线（孵蛋 · 换代 · 图鉴）
1. 照顾 / 完成离谱任务 → 赚 coins / 蛋。
2. 蛋孵化 → AI 现编全新宠物（外形+名字+身世+性格，稀有度决定离谱度）。
3. 同时只重点养一只。想换新的时，当前主宠**毕业**：DeepSeek 据其人设+进化史+名场面写沙雕生平传记，进图鉴，新宠上位。
4. **图鉴**是"你养过的所有奇葩编年史"，可随时重温传记（只读，不可再聊天）。

### 每日循环
打开(看衰减后状态+被吐槽) → 照顾补状态 → 唠嗑图乐 → 完成离谱任务赚币/蛋 → 日积月累触发变异惊喜 → 攒够孵新蛋换代 → 循环重开。

## 6. AI / DeepSeek 集成

### DeepSeek 的 7 类调用
| 活 | 频率 | 成本 | 输出 |
|---|---|---|---|
| 孵化生成人设 | 低 | 中 | JSON |
| **聊天回复** | **高** | 主要开销 | 流式文本 |
| 进化事件文案 | 低 | 中 | JSON/文本 |
| 毕业传记 | 很低 | 中 | 文本 |
| 离谱任务生成 | 低 | 低 | JSON（可批量缓存） |
| 特殊反应 | 偶尔 | 低 | 文本 |
| 记忆压缩 | 低 | 低 | 文本 |

### Prompt 架构（性格引擎）
每次聊天现拼 system prompt：`人设(persona) + 当前状态(stats) + 记得的事(memory_summary) + 安全护栏`，后接**最近 N 条**对话（更久靠 memory_summary 兜底）→ 上下文有界。回复**流式返回**做打字机，`max_tokens` 压短。

### 关键取舍：聊天效果来源
**聊天走流式纯文本（体验优先），游戏效果交给确定性规则引擎**（聊天 = +亲密度、关键词检测影响进化倾向）。只有孵化/进化/传记/任务这些需要结构化的活才用 JSON 输出。避免 AI 瞎改数值。

### 省钱护栏
- 模型用 `deepseek-chat`（V3），不用推理模型。
- 按钮反应本地模板化。
- `max_tokens` 压短 + 滚动记忆 → 上下文有界。
- **每用户限流**（每分钟 X 条）。
- **每用户每日 AI 聊天软上限 ~50 条**，到顶用人设文案"今天聊累了"挡一下，防公开后被刷爆账单。

### 安全护栏
system prompt 写死："无厘头可以，但不碰政治敏感/黄暴/辱骂真人/危险教程；被问到就歪头装傻岔开。" 对玩家输入加一层轻过滤。够沙雕但不越界。

### 失败兜底
- 聊天失败 → 符合人设的卖萌文案 + 退避重试；**失败不改数值**。
- 结构化任务失败 → 校验 JSON，严格重试一次，仍坏则**降级预置模板**。**孵化绝不允许出现"孵不出宠物"的死局**。
- 限流/额度到顶 → 友好人设文案挡，不报错。

## 7. 界面与交互

移动端竖屏单页。画面不追求精致，靠 emoji/符号 + 小动画 + 大字号沙雕文案撑氛围。

### 屏幕结构（底部三 tab：🏠家 / 📖图鉴 / 🥚蛋）
- **🏠 家（核心）**：顶部状态条 + coins；中间宠物本体（idle 动画）+ 头顶**碎碎念气泡**（活着感关键，多为本地模板、偶尔 AI）；任务条显示今日离谱要求；底部照顾按钮 + 💬聊天入口。
- **💬 聊天**：从家滑出的聊天面板，打字机流式回复。
- **📖 图鉴**：毕业老宠卡片网格，点开看沙雕生平传记 + 最终形态 + 名场面（只读）。
- **🥚 蛋/孵化**：看蛋与稀有度，孵化走悬念揭晓动画 → 蹦出 AI 新宠人设卡 → 确认换代。
- **⚡ 进化事件**：全屏浮层演夸张过场，演完新形态回到家。

### 新手引导
"路边捡到一颗会抖的蛋" → 孵化 → 生成第一只宠物 → 落到家界面。**30 秒内见到第一只沙雕宠物**。

### 交互手感
碎碎念气泡撑活着感；聊天打字机流式；按钮即时反馈（动画+状态条动+反应气泡）；离线回来数值衰减 + 宠物阴阳怪气吐槽。

### 视觉基调
**复古电子宠物机 + 表情包**混搭：干净、俏皮、大字号、meme 感。基调可在实现阶段微调。

## 8. 错误处理与测试

### 错误处理
- **AI 失败**：见 §6 失败兜底。
- **前后端交互**：按钮走**乐观更新**，失败回滚+轻提示；网络超时重试+温和提示。
- **数据一致性**：衰减服务端懒计算，**服务端唯一真相源**；效果结算走**数据库事务**；发奖励操作（孵蛋、领任务奖励）做**幂等**防双发。
- **账号**：JWT 过期静默刷新或温和引导重登。

### 测试策略
- **后端（重点）**：
  - **规则引擎单元测试**（价值最高）：衰减数学、进化触发、奖励/经济、幂等。
  - **接口测试**：账号、CRUD，用测试库。
  - **DeepSeek 全程 Mock**，CI 不调真 API；测 prompt 拼装 + 响应解析/降级兜底（专门喂坏 JSON）。
- **前端**：关键流程（状态条、聊天、孵化）组件测试，后端 Mock，保持轻量。
- **AI 质量**：搞一个 **prompt 试验台**——固定场景改 prompt 后拿真 API 重跑，肉眼看够不够骚/越不越界。真正的"好玩测试"靠真人试玩 + 打日志调 prompt。

## 9. 非目标（本版明确不做，YAGNI）

- 第三方登录（微信/Google）——用简单账号即可。
- 多宠同时养 / 宠物小队切换——保持"一只主宠"。
- 图鉴老宠可再聊天——本版只读。
- 变现 / 内购 / 广告。
- 精致美术 / 自绘立绘——emoji + 文案为主。
- 后台定时任务——数值一律懒计算。

## 10. 升级路径（留好的位）

- 账号系统可扩展到第三方登录。
- 图鉴老宠"回聊"功能。
- 缓存层 / 更强限流应对流量增长。
- 更丰富的道具 / 经济 / 社交（分享、炫耀、对战）。
